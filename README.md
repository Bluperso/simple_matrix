# Реализация библиотеки для работы с матрицами (matrix)

## Описание
Проект - реализация библиотеки для работы с матрицами на языке C. Библиотека предоставляет базовые операции над матрицами (сложение, вычитание, умножение и т.д.), а также чуть сложные алгоритмы: вычисление определителя, обратной матрицы, матрицы алгебраических дополнений.

## Возможности
Поддерживаются следующие операции:
- Создание и удаление матриц (`create_matrix`, `remove_matrix`)
- Сравнение матриц с заданной точностью (`eq_matrix`)
- Сложение матриц (`sum_matrix`)
- Вычитание матриц (`sub_matrix`)
- Умножение матрицы на число (`mult_number`)
- Умножение двух матриц (`mult_matrix`)
- Транспонирование матрицы (`transpose`)
- Вычисление определителя (`determinant`)
- Вычисление матрицы алгебраических дополнений (`calc_complements`)
- Нахождение обратной матрицы (`inverse_matrix`)

Все функции возвращают коды ошибок в соответствии с общими соглашениями:
- `0` - операция выполнена успешно
- `1` - некорректная матрица (INCORRECT_MATRIX)
- `2` - ошибка вычисления (CALC_ERROR), например, несовпадение размеров для сложения/умножения, вырожденная матрица для обратной и т.п.

При сравнении матриц используется точность `1e-6`.

## Требования
- Компилятор с поддержкой стандарта C11 (например, `GCC`)
- Библиотека `check` для запуска тестов (опционально, для `make test`)

## Сборка и тестирование
Для сборки статической библиотеки выполните:
```
make
make test
make clean
make gcov_report - генерирует отчет по покрытию строк в библиотеке
make 
```

## Структура проекта
```

│   └── tests.c       # модульные тесты (с использованием check)
```

## Примечания
- Все матрицы хранятся в виде двумерных массивов `double`.
- При сравнении используется абсолютная погрешность `1e-6`.
- Для вычисления определителя и обратной матрицы реализован метод приведения к треугольному виду (метод Гаусса) с выбором главного элемента по столбцу.

## Тестирование и покрытие
- Модульные тесты написаны с использованием фреймворка `check`. Они покрывают все основные функции
- Цель `test` так же генирирует отчет valgrind о возможных утечках памяти. Отчет будет в файле valgrind_report.txt
- Проверка покрытия по строкам производится с помощью цели `make gcov_report`.
- Процент покрытия (99.1%) можно увидеть в консоли или запускить файл coverage_report/index.html с помощью браузера, что бы увидеть
более подробную информаци.

- Для запуска тестов требуется установленная библиотека `check`.
